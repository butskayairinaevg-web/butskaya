Control Chart (SPC / Shewhart Chart)

A Control Chart is a statistical tool used to monitor, control, and improve a process over time by plotting data in time order and comparing it with statistically calculated control limits.
It is a core tool of Statistical Process Control (SPC) and was developed by Dr. Walter A. Shewhart at Bell Laboratories.

Also called: Shewhart Chart, Process Behaviour Chart, SPC Chart
Usage: 7 QC Tools · 8D · PDCA · Six Sigma

Why use a Control Chart?

A Control Chart separates normal variation (common causes) from abnormal variation (special causes), helping you decide when to act and when to leave the process alone.

Purpose of a Control Chart

Why we use it

Control Charts are used wherever a process produces data over time and you want to manage variation using statistical evidence instead of guesswork.

  • Monitor process variation and performance over time.
  • Detect special cause variations that require investigation.
  • Maintain process stability and predictability.
  • Identify when corrective actions are really necessary.
  • Reduce defects, scrap, and rework and improve quality.
  • Predict future performance and support capability analysis (Cp, Cpk, Pp, Ppk).
When used correctly, a Control Chart becomes one of the most powerful tools for quality control, continuous improvement, and decision making.

Key Components

Structure of the chart

Every Control Chart has a common structure that makes interpretation simple and consistent.

Graph layout

  • Horizontal axis (X-axis): Sample number, time, or subgroup index.
  • Vertical axis (Y-axis): Sample statistic (individual value, subgroup mean X̄, range R, standard deviation S, or defect counts).
  • Data points: Actual process measurements or subgroup statistics plotted in sequence.

Statistical lines

  • Centre Line (CL): Process average or mean (target / X̄).
  • Upper Control Limit (UCL): Mean + 3σ (or A₂×R̄ etc., depending on chart).
  • Lower Control Limit (LCL): Mean − 3σ (or A₂×R̄ etc., depending on chart).
  • Control rules: Patterns and limits used to detect abnormal behaviour.

Types of Control Charts

Variables & attributes

Control Charts are broadly divided into charts for variable data (continuous measurements) and attribute data (counts of defects/defectives).

A. Variable Control Charts

Used when the quality characteristic is measured on a continuous scale (mm, kg, seconds, bar, etc.).

  • X̄ – R Chart: For subgrouped data with small n (typically 2–10); monitors mean and range.
  • X̄ – S Chart: For larger subgroups (n ≥ 10); monitors mean and standard deviation.
  • I – MR Chart: Individuals and Moving Range chart for single observations (n = 1).

B. Attribute Control Charts

Used for count data such as number of defects or defective units.

  • p Chart: Proportion defective (fraction defective) with varying sample size.
  • np Chart: Number of defectives with constant sample size.
  • c Chart: Count of defects for a constant inspection area or unit.
  • u Chart: Defects per unit when sample size or area varies.

Control Chart Interpretation Rules

In-control vs out-of-control

A process is considered out of control when the plot shows evidence of special causes instead of only random variation.

  • Any single point outside the UCL or LCL.
  • Seven (or more) consecutive points on one side of the mean.
  • Continuous upward or downward trends.
  • Repeating or cyclic patterns, or “irrational” behaviour that cannot be random.
If any of these signals appear, investigate root causes (machine, tooling, material, method, operator, environment) before taking action on the process.

Control Chart vs Run Chart

Key differences

Both Run Charts and Control Charts show data over time, but only Control Charts include statistically calculated control limits and rules for detecting special causes.

Feature Control Chart Run Chart
Control limits (UCL & LCL) Yes – statistically calculated from data No – typically only a centre/median line
Detects special causes Yes – uses formal rules and 3σ limits Limited – mainly visual trends and shifts
Statistical basis Strong – based on probability and sampling Weak – descriptive only
Primary purpose Process control and stability Trend and shift monitoring
Control Charts are also known as Shewhart Charts because they were introduced by Dr. Walter A. Shewhart in the 1920s as part of Statistical Process Control.

When to Use a Control Chart

Practical scenarios

Use a Control Chart whenever you collect data in time order and want to distinguish normal variation from abnormal issues.

Typical situations

  • Process produces data over time (daily dimensions, hourly defect count, batch hardness, plating thickness, etc.).
  • You need to separate common cause variation from special cause variation.
  • Before adjusting machine settings (to avoid tampering when process is actually stable).
  • During commissioning of a new process or new machine to confirm stability.
  • To monitor Critical-to-Quality (CTQ) characteristics affecting function, safety, compliance, or customer satisfaction.

Projects & compliance

  • Measure, Analyse, and Control phases in Six Sigma DMAIC and Kaizen projects.
  • Early warning before parts go out of specification.
  • Evidence of statistical control for audits (ISO / IATF 16949), PPAP, and customer requirements.
  • To reduce rejection, rework, and improve process capability (Cp, Cpk, Pp, Ppk).
Continuous data → X̄–R, X̄–S, I–MR Attribute data → p, np, c, u

Benefits of Control Charts

Technical & operational

Control Charts provide both technical insight into variation and practical benefits in production and service environments.

Technical benefits

  • Early detection of small shifts, trends, and abnormal patterns.
  • Clear distinction between normal (common) and abnormal (special) variation.
  • Improved process stability, predictability, and capability (Cp, Cpk, Pp, Ppk).
  • Essential input for Root Cause Analysis (RCA) by showing when a problem started and whether it is temporary or permanent.

Operational benefits

  • Reduced defects, scrap, rework, and customer complaints.
  • Better productivity and machine efficiency by controlling drift and misalignment.
  • Data-based decision making instead of assumptions or over-adjustment.
  • Better communication between operators, inspectors, engineers, and managers using a simple visual tool.
  • Support for continuous improvement (Kaizen, Lean, Six Sigma) and compliance (ISO, IATF, PPAP).
A stable process with a well-maintained Control Chart delivers consistent quality and higher customer satisfaction with lower Cost of Poor Quality (COPQ).

Steps to Create a Control Chart

Practical workflow

The same high-level procedure applies to most Control Charts; only formulas and constants change with chart type and data type.

  1. Select the quality characteristic (CTQ)
    Choose the parameter to monitor (dimension, weight, thickness, pressure, number of defects or defectives, etc.).
  2. Choose the correct chart type
    Variables: X̄–R, X̄–S, or I–MR. Attributes: p, np, c, or u chart.
  3. Plan data collection
    Decide sample size (n), frequency (hourly, per shift, per batch), measuring instrument, and responsible operator (ensure MSA-compliant method).
  4. Collect data over time
    Record data in sequence, usually 20–25 subgroups for initial analysis.
  5. Calculate subgroup statistics
    For each subgroup calculate X̄, R or S, or for attribute charts calculate counts and proportions.
  6. Calculate control limits (UCL & LCL)
    Use standard SPC formulas for the selected chart (examples below).
  7. Plot the chart
    Draw CL, UCL, LCL and plot data points versus time/subgroup number.
  8. Interpret the chart
    Look for points outside limits, trends, runs, cycles, and sudden level shifts.
  9. Investigate special causes
    When abnormal patterns appear, perform root cause analysis and implement corrective actions.
  10. Maintain and update
    Keep adding new data, monitor stability, and revise limits when the process changes significantly.
Think of the Control Chart as a “live document” used daily in production and quality meetings, not a one-time study.

Calculation of Trial Control Limits

SPC formulas

Trial Control Limits (TCLs) are calculated from an initial dataset (typically 20–25 subgroups) and then refined after removing special causes to get final limits.

1. X̄ – R Chart (variables)

For k subgroups, each of size n:

For each subgroup i:
X̄ᵢ = Σx / n
Rᵢ = max(x) − min(x)

Grand mean: X̄̄ = average of all X̄ᵢ
Average range: R̄ = average of all Rᵢ

X̄ Chart:
UCLₓ̄ = X̄̄ + A₂ × R̄
CLₓ̄ = X̄̄
LCLₓ̄ = X̄̄ − A₂ × R̄

R Chart:
UCLᵣ = D₄ × R̄
CLᵣ = R̄
LCLᵣ = D₃ × R̄

Typical constants:

  • For n = 5: A₂ = 0.577, D₃ = 0, D₄ = 2.115
  • For n = 4: A₂ = 0.729, D₃ = 0, D₄ = 2.282

2. X̄ – S Chart (variables)

Mean chart:
UCLₓ̄ = X̄̄ + A₃ × S̄
CLₓ̄ = X̄̄
LCLₓ̄ = X̄̄ − A₃ × S̄

S chart:
UCLₛ = B₄ × S̄
CLₛ = S̄
LCLₛ = B₃ × S̄

Example constants for n = 10: A₃ = 0.308, B₃ = 0.552, B₄ = 1.448.

3. p Chart (proportion defective)

For each sample i:
pᵢ = number of defectives / sample size nᵢ

Average fraction defective:
p̄ = Σpᵢ / k

For sample size n:
UCL = p̄ + 3 × √[p̄(1 − p̄) / n]
CL = p̄
LCL = p̄ − 3 × √[p̄(1 − p̄) / n] (if LCL < 0, set LCL = 0)

4. np Chart (number of defectives)

For constant sample size n:
np̄ = average number of defectives
p̄ = np̄ / n

UCL = np̄ + 3 × √[np̄ (1 − p̄)]
CL = np̄
LCL = np̄ − 3 × √[np̄ (1 − p̄)]

5. c Chart (count of defects)

c̄ = average number of defects per sample

UCL = c̄ + 3 √c̄
CL = c̄
LCL = c̄ − 3 √c̄ (if LCL < 0, set LCL = 0)

6. u Chart (defects per unit)

For each sample i:
uᵢ = cᵢ / nᵢ (defects per unit)

ū = Σuᵢ / k

UCL = ū + 3 × √(ū / nᵢ)
CL = ū
LCL = ū − 3 × √(ū / nᵢ)
After you plot trial limits, remove points with known special causes, recalculate limits, and then use the revised limits for ongoing process control.
